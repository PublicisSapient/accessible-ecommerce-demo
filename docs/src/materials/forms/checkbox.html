--- notes: | The real **magic** of the styled form components is mostly in the CSS: * The `input` element is given `opacity: 0` so that it can't be seen, but is still an active element on the page. * The `label` and the `input` are connected via `for` and `id` attributes, which allows the input to be triggered by clicking on the label * The new checkbox UI is created with the `::before` and `::after` psuedo elements on the `label` * The `:focus` and `:checked` states on the `input` are passed to the `label` with the sibling selector (`input:checked + label`), so it **important to note** that the `input` element must preceed the `label`. > **Accessibility:** > > Groups of `checkbox` inputs should be wrapped in a `fieldset` and have a `legend`, which can be hidden from sighted users. ---
